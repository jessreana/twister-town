# 🎮 The Build-A-Game Challenge

A head-to-head contest: each of us builds a browser game using Claude, then we swap and play each other's.

---

## The Challenge

Each player picks their **own** game and builds it as a single self-contained HTML file that runs by just opening it in a browser. No setup, no server.

Pick something you can realistically finish — snake, pong, a maze, whack-a-mole, a quiz game, tic-tac-toe vs. the computer, a typing tester, a clicker game, whatever you think you can make shine in the time limit. Ambition is allowed, but unfinished doesn't score well.

## Every game must have

- A clear way to start and restart without refreshing the page
- A win/lose state or a score — something that makes it a *game*, not a toy
- Instructions visible somewhere so the other person can pick it up cold
- It has to actually run and be playable start to finish

## Rules

- ⏱️ **50 minutes** on the clock
- Prompt Claude as much as you want, but **no editing the code by hand**
- Keep your game choice **secret** until the reveal
- At the end, swap files and play each other's

---

## Scoring

Each player scores the other's game 1–10 in three categories:

| Category | What it means | Score |
|---|---|---|
| **Fun** | Did you actually want to keep playing? | /10 |
| **Polish** | Does it look and feel finished? | /10 |
| **Ambition** | How much did they bite off — and did they land it? | /10 |
| | **TOTAL** | /30 |

Highest total wins. Simple-but-flawless can beat huge-but-janky — that's the point.

Good luck. May the better prompter win. 🏆
